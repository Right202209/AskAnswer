  总体评价

  设计扎实、范围克制。三件事处理得尤其好：

  1. 不动 reducer — upsert_meta 在 cli.stream_query 末尾写而不是改图状态，避开
  add_messages 之类的合并语义。
  2. HITL 兼容 — 阶段 B 明确 "pending_shell 非空时拒绝 /undo"，这正是
  _run_with_confirmation (_react_internals.py:239) 的硬约束。
  3. token 计量用 callback 而非改 _ModelProxy — 切模型时不会丢失 hook（前提是
  set_model 也要 re-attach，见下方风险 6）。

  计划里需要补强的 11 处

  阶段 A 关键

  1. 删 thread 时不只删 checkpoints 表。SqliteSaver 实际写三张表：checkpoints、w
  rites、checkpoint_blobs（langgraph-checkpoint-sqlite ≥ 2.0）。计划里 DELETE 
  FROM checkpoints WHERE thread_id=? 会留下孤儿 blobs / writes。需要在事务里
  DELETE 三张表 + thread_meta。
  2. WAL 模式必须开。计划没提并发 — 但用户开两个终端跑 askanswer
  是常见场景。PersistenceManager.__init__ 必须执行：
  PRAGMA journal_mode=WAL;
  PRAGMA busy_timeout=5000;
  2. 否则两个进程一起写就 database is locked。
  3. --graph 模式不应该初始化持久化。当前 cli.export_graph (cli.py:714) →
  draw_search_assistant_mermaid() → create_search_assistant() 会一并触发
  get_persistence()，导致 --graph 也在 ~/.askanswer/state.db 创建空 DB
  文件。建议：让 create_search_assistant(checkpointer=None)
  接可选参数，draw_search_assistant_mermaid 显式传 None。
  4. schema 迁移机制必须从阶段 A 就铺好。thread_meta 现在 8 列，阶段 C 的
  audit_event 又要加表 — 不加 PRAGMA user_version 或 schema_version
  表，第二次发版改列就尴尬。建议阶段 A PR 就引入 _migrate(conn, 
  target_version=1) 骨架，后续阶段只加 _migrations[2] = ...。
  5. /resume 在挂起 thread 上的语义没说清。如果用户 /resume 一个上次 interrupt()
   没 resume 完的 thread，下一条 HumanMessage 会被 langgraph 当成新输入，挂起的
  pending_shell 就僵在那。建议 /resume 时检查
  app.get_state(config).tasks[*].interrupts，非空时给两个选项："恢复挂起的 shell
   确认 / 放弃挂起开新一轮"。
  6. set_model 必须保留 audit callback。_ModelProxy._swap (load.py:52) 替换
  _inner 时会丢掉手动 attach 的 handler。阶段 C 的正确做法是在 _build_model
  里默认注入 audit handler，而不是 attach 到现成 backend，这样 set_model 走
  _build_model 自动带上。

  阶段 A 体验

  7. /clear 不再清旧 thread 是用户可见行为变化。建议 /clear
  输出明确提示："上一段对话已保留为 thread <id>，可用 /threads 查看 / /delete
  删除"。否则用户会以为 /clear 没生效。
  8. 斜线命令命名空间快爆了。计划新增 12 个命令（当前才 6 个）。建议归类成
  /thread list|resume|title|delete、/cp list|undo|fork、/audit 
  show|export|usage，避免一级命令膨胀也利于阶段 C 加更多。

  阶段 C/D 细节

  9. pricing.py 的"陈旧风险"。OpenAI 价格半年一改。建议：(a)
  文件顶部写最近更新日期；(b) 允许 ~/.askanswer/config.toml 的 [pricing]
  段覆盖；(c) 未知模型只记 token 不记成本（计划已写对）。
  10. /import 不应恢复 audit_event。计划阶段 C 的 /export --format json 包含
  audit，但 import 时若一并恢复，等于让审计可被伪造。建议 import 只恢复 messages
   + meta，并在新 thread 写一条 kind=imported 的 audit。
  11. 阶段 D 加密边界要画清。SqliteSaver 不只写 checkpoint，还有 writes（增量
  ToolMessage）。只加密 checkpoint 等于没加密 — writes
  里的查询参数和工具结果照样泄露。如果走"应用层 Fernet"路线，包装层必须覆盖
  BaseCheckpointSaver.put / put_writes 两条路径。
