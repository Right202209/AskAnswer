⏺ AskAnswer

基于 LangGraph 的命令行智能助手

按意图分流到「读文件 / 联网搜索 / 数据库 SQL / 数学计算 / Helix 规格演化 / 直接回答」六类，由 LLM 在 ReAct 子图里调度工具，支持 HITL Shell 确认、自评改写与 Markdown 渲染。

快速开始 GitHub

核心特性

🧭 意图分流

自动识别 file_read / search / sql / math / helix / chat 六类，本地正则优先、LLM 兜底。

🔧 工具调度

天气、时间、计算、汇率、IP、读文件、联网搜索、shell 等内置工具按 intent 暴露给模型。

🗄️ SQL Agent 子图

sql_query 工具内部跑 list_tables → schema → generate → check → run；DML/DDL 在执行前硬拦截。

🧬 Helix 规格演化

需求模糊时自动进入苏格拉底式访谈 → seed → execute → evaluate 多代演化，避免「想做但没想清楚」直接动手。

🌐 MCP 热接入

SSE / streamable-HTTP / stdio 三种传输，运行时 /mcp <url> 即连。默认仅在 chat intent 暴露，可设 ASKANSWER_MCP_ALL_INTENTS=1 放开到所有 intent。

🪞 自评改写

sorcery 节点按 intent 分派 evaluate；search 路径允许改写关键词重跑，最多 2 次。

💾 SQLite 持久化

LangGraph checkpoint + 自管 thread_meta 共用一个 SQLite 文件；/threads / /resume / /title / /delete 管理历史会话。

🖥️ 类 Claude REPL

prompt_toolkit 输入、补全、续行；箱式 UI、⏺ Node(detail) 进度、Markdown 渲染、双 Ctrl-C 退出。

⚡ Shell 前缀（HITL）

!cmd 直接执行；ReAct 内的 gen_shell_commands_run 走人机确认。命中高风险黑名单（rm/mkfs/dd/sudo/...）会拦截。

安装 & 配置

pip install -e .

在项目根目录创建 .env：

OPENAI_API_KEY=...
TAVILY_API_KEY=...
OPENWEATHER_API_KEY=...                 # 可选，启用天气工具
ASKANSWER_DB_PATH=~/.askanswer/state.db # 可选，覆盖默认 SQLite 路径
ASKANSWER_MCP_ALL_INTENTS=1             # 可选，把 MCP 工具放开到所有 intent

askanswer

安全须知

  • Shell 前缀：!cmd 与 gen_shell_commands_run 都用同一份高风险黑名单（rm / rmdir / shutdown / reboot / mkfs / dd / sudo / fork-bomb / 重定向覆盖 / kill -9 / chmod -R / chown -R / mv 到根 / 写磁盘设备）；命中后必须二次确认。黑名单 != 白名单，恶意构造仍可能绕过，请只在可信环境使用。
  • MCP：连接的 MCP server 完全决定它暴露什么工具，工具描述会被注入到模型的 system prompt。请只接信任的 server；默认仅 chat intent 可见，需要时再 ASKANSWER_MCP_ALL_INTENTS=1 放开。
  • SQL：DML/DDL 在 sql_db_query 执行前硬拦截。如仍需写库，请直接走数据库自身的工具，不要绕过本系统。
  • 读文件：会拒绝读取 .env / SSH 私钥 / credentials / .pgpass 等敏感文件名，以及超过 10 MB 的文件。可通过 ASKANSWER_READ_FILE_MAX_BYTES 调整上限。
  • 历史：prompt_toolkit history 会过滤含 API key / token / 嵌入凭据的 DSN 的输入；ASKANSWER_DB_PATH 同目录下生成 history 文件，敏感会话建议手动清空。
  • .env 覆盖：load_dotenv(override=True) 让仓库内 .env 覆盖 shell 环境变量。在不可信目录运行前请检查 .env 内容。
