# Architecture ‚Äî Runtime services (MCP / persistence / CLI / runner / HTTP server / model)

## Runner (`askanswer/runner.py`) ‚Äî UI-free event stream

`runner.py` is the front-end-agnostic core extracted ahead of the C1 cli split: `stream_leg(app, graph_input, *, config, context)` yields typed `RunEvent`s (`token` / `tool` / `node` / `interrupt` / `final`) for one **leg** ‚Äî from one graph input (fresh query or `Command(resume=...)`) to the next interrupt or completion. Event semantics mirror `cli.stream_query` line-for-line (AIMessageChunk-only tokens, one `tool` event per planning phase = consumer resets its token buffer, `__interrupt__` capture + `state.tasks` fallback, final-answer fallback from state). `run_leg(...)` wraps it with the CLI-finally bookkeeping (audit `begin_run`/`flush_pending`/`end_run`, telemetry root span, `upsert_meta`; resume legs pass `preview=None` so COALESCE keeps the old preview) ‚Äî closing the generator early (client disconnect) still runs the finally. `runtime_context_from_env()` is the single env‚Üí`ContextSchema` mapping; `cli.stream._runtime_context` delegates to it. **Since C1, `cli.stream.stream_query` consumes `runner.stream_leg` directly (no second event loop): the runner is the single event-stream implementation for both CLI and HTTP ‚Äî streaming-semantics changes live in `runner.py` alone.** The CLI keeps its own turn-level bookkeeping (audit `begin_run`/`end_run`, telemetry root span, `upsert_meta`) around the `stream_leg` legs rather than delegating to `run_leg`, so multi-leg audit granularity matches pre-split behavior.

## HTTP/SSE server (`askanswer/server.py`, `askanswer/wire.py`)

Stdlib-only (`ThreadingHTTPServer`, zero new deps), started via `python -m askanswer.server` (or `askanswer-server` after reinstall). Endpoints: `GET /health`, `GET /v1/interrupt?thread_id=`, `POST /v1/query {query, thread_id?}` and `POST /v1/resume {thread_id, decision}` ‚Üí SSE streams (`meta` ‚Üí token/tool/node/interrupt ‚Üí final? ‚Üí `done{status}`). HITL over HTTP: the stream ends at `interrupt` with `done.status=interrupted` (checkpointer persists the pause); the client later POSTs `/v1/resume` with the same decision shapes the CLI uses. One shared lazily-built `create_search_assistant()` (import of the module touches neither graph nor persistence). Guards: optional `ASKANSWER_SERVER_TOKEN` bearer auth (compare_digest), non-localhost `Origin` ‚Üí 403 + JSON-only content type (CSRF), 64KB body cap, `BoundedSemaphore(2)` run slots (503), per-thread busy set (409), 60s socket timeout. `wire.py` holds transport helpers: `sse_frame`, `event_wire` (node events ‚Üí scalar-only summaries; interrupt payloads ‚Üí recursive `json_safe`), `RequestError`, `split_path`/`is_local_origin`/`normalize_thread_id` (reused by C4's JSON endpoints later). Verification checklist (not yet run): `docs/important-documentation-c3-http-sse-server.md`.

## MCP (`askanswer/mcp.py`)

`MCPClientManager` runs a dedicated asyncio loop on a background thread. All `add_url`/`add_stdio`/`call_tool` calls hop onto that loop via `asyncio.run_coroutine_threadsafe` so each server's async context manager enters and exits in the same task ‚Äî necessary to avoid anyio cancel-scope errors. Always go through `get_manager()` / `shutdown_manager()`; the latter is registered in `cli.main` with `atexit`. Tool specs returned by `list_tools()` are wrapped into `StructuredTool`s by `registry._wrap_mcp_tool` (which converts the JSON Schema to a flat pydantic model ‚Äî non-trivial schemas with `$ref`/`anyOf` are skipped).

Each `_ServerEntry` tracks `status` / `last_checked` / `last_error`. `health_check(name=None)` probes a server (or all) with a 3s-timeout `list_tools()` on the background loop, updates those fields, and refreshes the tool cache; `registry.refresh_mcp()` then skips `disconnected` servers so dead tools disappear from the LLM's view. Connections persist to an autoconnect **profile** (`askanswer/mcp_profile.py`, `~/.askanswer/mcp.json` or `ASKANSWER_MCP_PROFILE`, atomic temp-file+`os.replace` writes): `/mcp <url>` and `/mcp add_stdio` append, `/mcp remove` deletes, and `cli._autoconnect_mcp_profile()` (called in `main` after registry seed) replays them on startup ‚Äî per-server failures only warn. `mcp_profile.py` deliberately imports nothing from `mcp`/`registry`/`graph`.

## Persistence (`askanswer/persistence.py`)

A single SQLite file (default `~/.askanswer/state.db`, override with `ASKANSWER_DB_PATH` or `XDG_DATA_HOME`) carries both LangGraph's `SqliteSaver` checkpoint tables and our `thread_meta` table (title, preview, last_intent, model_label, created_at, updated_at, message_count, tags, tenant_id). Both share one `sqlite3.Connection` opened with `check_same_thread=False`, WAL journal mode, `busy_timeout=5000`, and `synchronous=NORMAL` so two concurrent `askanswer` processes don't immediately hit `database is locked`. Schema versioning lives in our own `askanswer_schema` row (**current version: `4`**); add a `if current < N` branch in `_migrate` when adding columns (`_add_column_if_missing` does an idempotent `ALTER TABLE` via `PRAGMA table_info`). v3 added `tenant_id` to `thread_meta`+`audit_event` (with per-tenant indexes); v4 added the `checkpoint_label` table. `get_persistence()` is a lazy singleton ‚Äî never construct `PersistenceManager` directly, and never import this module from `graph.py` at top level (the `--graph` path explicitly avoids creating the DB by passing `InMemorySaver`).

**Tenant isolation**: read methods (`list_threads` / `get_meta` / `find_by_prefix` / `delete_thread` / `list_audit_events` / `usage_summary`) take `tenant_id=None`; `None` = no filter (legacy behavior), a value filters via `tenant_id = ?`. Write methods (`upsert_meta` / `log_audit_event(s)`) stamp `tenant_id`, defaulting to `_default_tenant_id()` (`ASKANSWER_TENANT_ID` env ‚Üí `None`). `upsert_meta`'s UPSERT preserves original ownership with `COALESCE(thread_meta.tenant_id, excluded.tenant_id)`. The CLI passes `_current_tenant()` into every persistence call; `delete_thread` also re-checks ownership before touching any checkpoint tables. SqliteSaver's own tables are NOT tenant-columned ‚Äî isolation is enforced at the `thread_meta`/`audit_event` filter layer.

## CLI (`askanswer/cli/` package)

Since C1, `cli.py` is an `askanswer/cli/` package (each module ‚â§300 lines; `__init__.py` holds `main` + back-compat re-exports so `from askanswer.cli import main/stream_query/handle_command/...` still resolve). Module map: `theme` (rich `_console` + ANSI class `C`) / `text` (`_visual_width`, `_truncate`, `_format_ts`, `_term_width`) / `render` (welcome/status/help panels, `render_answer`/`render_error`) / `progress` (`‚è∫ Node` markers) / `confirm` (HITL menus) / `stream` (`stream_query`) / `repl` (`interactive_loop` + `!<cmd>`) / `app` (parser, `--graph` export, MCP autoconnect, telemetry init) / `commands/` (router `__init__` + one module per domain: `model` / `mcp` (+`mcp_view`) / `threads` / `timetravel` / `audit` / `transfer`, with shared `_common`). `stream_query` consumes `runner.stream_leg` (see Runner section) ‚Äî no per-node `app.stream` loop of its own anymore; it maps `token`‚Üírich.Live, `node`‚Üí`‚è∫` marker, `interrupt`‚Üí`confirm._prompt_confirmation`. `ui_input.py` owns input, `ui_spinner.py` owns the wait-state spinner. The intent label on the `understand` marker comes from `handler.cli_label(update)` via `progress._intent_cli_label` ‚Äî don't hard-code intent strings. Slash commands (`/help [cmd]`, `/clear`, `/status`, `/model`, `/mcp ‚Ä¶`, `/threads [keyword]`, `/resume`, `/title`, `/delete`, `/checkpoints`, `/undo`, `/jump`, `/fork`, `/audit`, `/usage`, `/export`, `/import`, `/exit`) route through `commands.handle_command`; `commands._common._LAST_LIST` (mutated via `remember_threads`/`forget_thread`) caches the most recent `/threads` output so `/resume 1` / `/delete 1` address rows by index. The single canonical command list lives in `ui_input.SLASH_COMMANDS` ‚Äî `render.help_block` and the completer both read it. The `!<cmd>` prefix bypasses the graph entirely and runs a shell command after a danger check.

## Input session (`askanswer/ui_input.py`)

Wraps `prompt_toolkit.PromptSession` with `FileHistory` at `~/.askanswer/history` (same dir as `state.db`), a slash-command `Completer` that reads `SLASH_COMMANDS`, a `bottom_toolbar` callback re-rendered on every keystroke (`thread:abc ¬∑ model:foo ¬∑ mcp:N`), and Emacs editing mode. Trailing backslash continues input on a new line ‚Äî `read_line` joins them with `\n`. Returns `None` for EOF or 2√ó Ctrl-C within 2s (caller exits), `""` for single Ctrl-C (caller redraws), text otherwise. The status callback takes a single-element list as a "mutable ref" for `thread_id` so `/clear` / `/resume` / `/delete` can swap it without rebuilding the session.

## Spinner (`askanswer/ui_spinner.py`)

Background-thread Braille spinner used during waits between node updates. `start()` / `stop()` are idempotent; `transition(text)` swaps the label and resets the elapsed-seconds counter so each "phase" is timed independently. `freeze_for(fn)` erases the spinner, runs `fn` (typically a marker print), then resumes ‚Äî keeps stdout from interleaving with the animation. No-op when `stdout` isn't a TTY.

## Streaming answer rendering

`stream_query` consumes both `updates` and `messages` channels. `_handle_message_chunk` filters `AIMessageChunk`s: chunks with `tool_call_chunks` (LLM emitting a tool call) flip `in_tool_phase` so the next user-facing tokens reset the buffer (avoids tool-routing chatter polluting the final answer). The first user-facing token stops the spinner and starts a `rich.live.Live(Padding(Markdown(...)))` panel that updates per token. When the parent `answer` node update arrives, the Live panel is replaced with the authoritative `final_answer` then closed (transient=False keeps the rendered content). `live_state["streamed"]` tells the tail of `stream_query` to skip `render_answer` (already on screen) or call it (model didn't stream / sorcery rewrote the answer).

## Terminal selector (`askanswer/ui_select.py`)

`select_option(options, *, prompt, default, free_input_label, free_input_prompt)` is the shared up/down menu. It uses `termios` + `tty.setcbreak` in TTY mode (‚Üë/‚Üì or k/j to navigate, Enter to confirm, Esc/Ctrl-C to cancel returning `CANCELLED == -1`, digits 1-9 to jump). When stdin/stdout is not a TTY it falls back to numbered text input. If `free_input_label` is set, it's appended as the last menu entry; selecting it triggers a free-text prompt and the typed string is returned as the second tuple element. Used by both the HITL shell consent and the Helix interview.

## Model loading & swap (`askanswer/load.py`)

`model` is a `_ModelProxy` wrapping the real `init_chat_model(...)` backend. Modules import it once at startup; `set_model("provider:name")` (called by `/model`) builds a new backend and `_swap`s it in place via `object.__setattr__`, so every existing reference is automatically redirected. **Always import `model` from `.load` and call through it** ‚Äî never cache the inner backend, or you'll keep talking to the old model after `/model`. `current_model_label()` returns the active `provider:spec` for diagnostics.

## Audit + cost (`askanswer/audit.py`, `askanswer/pricing.py`)

Telemetry is deliberately kept **out of `SearchState`**. `audit.py` collects events for one graph run via three `ContextVar`s (`_THREAD_ID`, `_PENDING`, `_TENANT_ID`); `stream_query` brackets each run with `begin_run(thread_id, tenant_id=‚Ä¶)` / `end_run(tokens)` / `flush_pending(...)` (tokens is now a 3-tuple) so events only get persisted to the `audit_event` SQLite table once at the CLI boundary, each stamped with the run's tenant. `LLMUsageCallback` is a `BaseCallbackHandler` attached via `with_llm_audit_callback(config, model_label=...)`; it pulls token usage out of the LLM response (`_extract_usage` walks several shapes ‚Äî OpenAI `usage`, LangChain `usage_metadata`, etc.). `pricing.py` is the cost table (USD per 1K tokens, per `provider:model` label) used by `/audit`; unknown labels return `None` so the report shows tokens without inventing a price.

## Observability exporters (`askanswer/telemetry/`)

A three-tier split: local `audit` (governance) stays independent; `telemetry/` adds external trace exporters with **independent write paths** but a shared minimal schema (kind / tool_name / duration_ms / tokens). `init_telemetry()` (called in `cli.main`) builds exporters only when gated env vars are set ‚Äî `LANGSMITH_API_KEY`/`LANGCHAIN_API_KEY` ‚Üí `telemetry/langsmith.py`, `ASKANSWER_OTEL_EXPORTER` ‚Üí `telemetry/otel.py`; SDK-missing ‚Üí the exporter's `build_*` returns `None`. When no exporter is active, `emit_event`/`span`/`open_span` are cheap no-ops (**zero client, zero overhead**). Trace context is a contextvar span stack (`_SPAN_STACK`), never `SearchState`. Wiring: `load.py` appends a `telemetry.llm_callback(...)` alongside the audit callback (lazy import, so all model calls trace); `stream_query` opens a root `askanswer.query` span; `_react_internals._emit_tool_telemetry` mirrors tool-call events. `_react_internals`/`load` import telemetry **lazily** (top-level-import ban) and never cache the `_ModelProxy`.

## Time travel (`askanswer/timetravel.py`)

`/checkpoints`, `/undo`, `/jump`, `/fork` are thin wrappers over LangGraph's `app.get_state_history(config)` + `app.update_state(...)`. `list_checkpoints` returns `CheckpointInfo` (index, node, step, message_count, created_at, `pending_confirm` flag, and `checkpoint_id`) for the most recent N snapshots; `rewind_to` rewrites the current thread to a prior checkpoint and returns a `RewindResult(target, affected_messages)`, refusing if `pending_confirmations`/`pending_shell` or a live interrupt is present; `fork_thread` clones a snapshot into a fresh `thread_id` (also seeds `thread_meta` so the new thread shows up in `/threads`). **Checkpoint labels**: `/undo [n] --label NAME` rewinds and records `NAME ‚Üí checkpoint_id` in the `checkpoint_label` table (`persistence.set_checkpoint_label`); `/undo --label NAME` (no index) resolves the label back to an index via `resolve_checkpoint_label` + `find_checkpoint_index_by_id`. `/checkpoints` shows a `label` column; `/undo` reports the affected-message count.
